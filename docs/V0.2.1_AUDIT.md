# Public Prototype v0.2.1 Audit

Audit date: 2026-07-21 (Asia/Kuala_Lumpur)
Baseline commit: `08db24a` on `main`
Production inspected: <https://selangor-mobility-navigator.pages.dev/>

## Scope inspected

- All tracked source, configuration, documentation, tests, Cloudflare Pages Functions, build scripts, and generated transit-data manifests were inventoried with `git ls-files`.
- `package.json`, `package-lock.json`, `wrangler.toml`, `.gitignore`, the complete single-page navigation in `index.html`, CSS, client modules, Pages Functions, tests, and public GTFS-derived assets were inspected.
- Generated schedule buckets were checked as a dataset family rather than treating each mechanically generated JSON shard as unique application logic.
- Production was loaded in a browser and its visible first view, attribution, runtime states, and console warnings/errors were inspected.

## Data provenance notes

- Transit static assets declare data.gov.my GTFS Static sources and carry processed timestamps in their manifests.
- Client realtime requests use Pages Functions for KTMB, Rapid KL bus, and MRT feeder feeds. These are source-backed vehicle positions, not proof of system-wide realtime coverage or “verified vehicles”.
- Schedule rows are derived from GTFS Static and must be presented as scheduled/static, not live arrivals.
- Weather responses are forecasts/warnings sourced through data.gov.my; they are not live observations or AQI.
- Fuel values are weekly Ministry of Finance/data.gov.my records and are not local station prices.

## Release blockers

1. Remove dummy payment/profile markup rather than hiding it at runtime.
2. Remove misleading Live/verified/live-pulse presentation.
3. Replace inert or fake-success interactions.
4. Add persistent Public Prototype disclosure and a useful Project/Data Status interface.
5. Convert unsupported account, fare, Park & Ride, alert, and feedback states into intentional Coming Soon/status experiences.
6. Align all release metadata to v0.2.1.
7. Validate navigation, responsive viewports, console, tests, build, Cloudflare preview, and production before tagging v0.2.1.
